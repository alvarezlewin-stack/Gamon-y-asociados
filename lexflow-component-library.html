Componentes de LexFlow

Cada componente documenta propósito, variantes, comportamiento y accesibilidad. Esta biblioteca es la referencia oficial para implementación — ningún componente nuevo debe crearse fuera de este sistema sin registrarlo aquí primero.

01

Botones

Propósito
Acción principal, secundaria o destructiva del usuario.
Variantes
Primary (dorado, relleno), Secondary (borde), Ghost (texto dorado), Danger (borde rojo), Disabled.
Tamaños
md (13px/10-18px padding) para acciones principales, sm (11.5px/6-12px) dentro de tarjetas o filas.
Accesibilidad
Contraste mínimo 4.5:1 texto/fondo; estado disabled no debe ser el único indicador — acompañar con `aria-disabled`.
Comportamiento
Hover: aclara 8%. Focus: anillo dorado de 2px. Nunca usar solo color para transmitir "primario" — el relleno lo distingue por forma.

03

Selectores

Propósito
Elegir entre opciones: dropdown, toggle de vista, checkbox, radio.
Variantes
Select nativo estilizado, Toggle de segmentos (Día/Semana/Mes), Checkbox, Radio.
Comportamiento
Toggle: un solo activo, cambia instantáneo sin confirmación. Checkbox/Radio: mínimo 16px de área táctil, 44px con padding en mobile.
04

Tablas

Propósito
Listar registros estructurados (procesos, documentos, clientes) con múltiples columnas.
Comportamiento
Fila resaltada sutil al hover (2% blanco). Encabezado en mono uppercase para diferenciarlo del contenido.
Responsive
En mobile (<700px) se reemplaza por tarjetas apiladas (ver componente Proceso), nunca scroll horizontal forzado.

Badges, etiquetas y chips

Badge
Relleno suave (`-dim`) + texto de color — para estado o severidad (riesgo, prioridad).
Tag
Solo borde, color por categoría — para materia jurídica (Civil/Laboral/Comercial/Familia).
Chip
Interactivo, para filtros — estado activo con borde y fondo dorado tenue.

10

Modales

Propósito
Interrumpir el flujo para una decisión puntual (confirmar, crear rápido).
Comportamiento
Overlay 50% negro, cierre con click afuera o Escape, foco atrapado dentro del modal (accesibilidad).
Tamaño
max-width 340-480px según contenido; nunca pantalla completa en desktop.

¿Eliminar esta actuación?

Esta acción no se puede deshacer. El registro se marcará como eliminado (soft delete).

Alertas y Toasts

Alerta
Persistente, dentro del flujo de la página (ej. "vencimiento próximo"). No se cierra sola.
Toast
Flotante, temporal (4-6s), confirma una acción ("Documento guardado"). Esquina inferior derecha en desktop, superior en mobile.

14

Buscador

Propósito
Búsqueda de texto libre dentro de listados (Procesos, Clientes, Documentos).
Diferencia con consulta IA
El buscador filtra datos existentes; la barra de consulta jurídica (Centro de IA) interpreta lenguaje natural — visualmente similares pero con ícono de color distinto (neutro vs. púrpura).

Propósito
Acotar un listado por categoría (materia, estado, prioridad).
Patrón
Chips horizontales con scroll en mobile, nunca dropdown de filtros en la primera capa — reduce toques.

19

Confirmaciones

Propósito
Variante de Modal para acciones irreversibles (eliminar, cerrar proceso).
Regla
El botón destructivo siempre a la derecha, nunca preseleccionado — evita eliminar por error de doble tap.

¿Cerrar este proceso?

El proceso pasará a "Archivado". Podrás reabrirlo desde el listado.

20

Indicadores de riesgo y prioridad

Origen del dato
Atributo manual definido por el abogado (Alta/Media/Baja — Alto/Medio/Bajo). La IA podrá sugerir un valor, pero no lo modifica automáticamente (decisión de dirección, ver DESIGN-ROADMAP D-014).
Semáforo
Mismo lenguaje de color en toda la app: rojo = alto, dorado = medio, verde = bajo. Un solo sistema, no uno por pantalla.

21

Componentes de IA

Principio
La IA nunca se disfraza de chat genérico. Se identifica con color púrpura y el "ai-dot" (punto con halo) en cualquier contexto donde aparezca.
Componentes
Badge de IA (etiqueta), Sugerencia (barra lateral púrpura + texto), Barra de consulta (buscador con ícono púrpura en vez de neutro).
Regla de contenido
Todo contenido de IA hoy es ilustrativo — no se presenta como dato real hasta que el modelo esté conectado.

Reglas responsive

Teléfono
<700px — 1 columna, sidebar oculto, tablas → tarjetas apiladas, chips con scroll horizontal.
Tablet
700–1080px — 2 columnas donde antes había 3-4, sidebar visible desde 900px.
Escritorio
>1080px — grillas completas (3-4 columnas), sidebar fijo de 220-240px. (Futuro — la app hoy es mobile-first.)
Unidad base
Espaciado en múltiplos de 4px (4/8/12/16/22/32) — ya aplicado en todos los componentes de esta biblioteca.
